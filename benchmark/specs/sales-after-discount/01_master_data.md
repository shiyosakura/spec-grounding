# BtoB販売管理システム 固定データ（マスターデータ）

> すべての画面・機能から参照される固定データ。運用中は管理画面から変更可能だが、見積/受注/出荷/請求処理の実行中は変化しない。
> 仕様書を書く際は必ずこのファイルを参照すること。
>
> 中小企業向け。取引先200社・商品1000点規模。
> 4モジュール（受注管理・在庫出荷管理・請求入金管理・マスター管理）・15画面構成。

---

| カテゴリ | データ名 | データ幅 | データの数 | データ説明 |
| :-- | :-- | :-- | :-- | :-- |
| 商品カテゴリ | カテゴリID | 0-99 | 1 | 0=未分類（予約値）。有効なIDは1以上 |
| | カテゴリ名 | 文字列 | 1 | UI表示用 |
| 商品マスター | 商品ID | 0-9999 | 1 | 0=なし（予約値）。有効なIDは1以上 |
| | 商品コード | 文字列 | 1 | 業務用の識別コード（例: PRD-001）。一意制約。検索対象 |
| | 商品名 | 文字列 | 1 | UI表示用。検索対象 |
| | カテゴリID | 0-99 | 1 | `商品カテゴリ`参照用のID |
| | 標準単価 | 0-9999999 | 1 | 円単位。`単価マスター`に特別単価がない場合のデフォルト |
| | 単位 | 文字列 | 1 | 個、箱、kg 等の取引単位 |
| | 有効フラグ | 0-1 | 1 | 0=無効（見積/受注の商品選択候補に表示しない）、1=有効 |
| 取引先マスター | 取引先ID | 0-9999 | 1 | 0=なし（予約値）。有効なIDは1以上 |
| | 取引先コード | 文字列 | 1 | 業務用の識別コード（例: CUS-001）。一意制約。検索対象 |
| | 取引先名 | 文字列 | 1 | UI表示用。検索対象 |
| | 住所 | 文字列 | 1 | 請求書・納品書の宛先 |
| | 電話番号 | 文字列 | 1 | |
| | メールアドレス | 文字列 | 1 | |
| | 締め日 | 0-28 | 1 | 月次請求の締め日。0=月末（センチネル値）。1〜28=該当日 |
| | 与信限度額 | 0-99999999 | 1 | 円単位。受注確定時の与信チェック閾値。0=与信チェックなし |
| 単価マスター | 単価マスターID | 0-9999 | 1 | 0=なし（予約値）。有効なIDは1以上 |
| | 取引先ID | 1-9999 | 1 | `取引先マスター`参照用のID |
| | 商品ID | 1-9999 | 1 | `商品マスター`参照用のID。取引先ID×商品IDの組合せで一意 |
| | 特別単価 | 0-9999999 | 1 | 円単位。この取引先に対する特別価格。`商品マスター.標準単価`より優先 |
| システム設定 | 消費税率 | 0-100 | 1 | %単位。デフォルト10。税額計算式: `合計金額 × 消費税率 ÷ 100` |
| | 見積有効日数 | 1-365 | 1 | 見積作成時のデフォルト有効期限日数。デフォルト30 |
| | 見積番号プレフィックス | 文字列 | 1 | 見積番号の接頭辞（例: EST-） |
| | 受注番号プレフィックス | 文字列 | 1 | 受注番号の接頭辞（例: ORD-） |
| | 請求番号プレフィックス | 文字列 | 1 | 請求番号の接頭辞（例: INV-） |
| | 出荷指示番号プレフィックス | 文字列 | 1 | 出荷指示番号の接頭辞（例: SHP-） |
